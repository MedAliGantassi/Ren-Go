RENGO BACKEND - UPDATED DOCUMENTATION
=====================================

1) BUSINESS LOGIC UPDATES
-------------------------

1.1 Messaging Rules
- Only authenticated users can send/read messages.
- ADMIN cannot send messages and cannot be used as message receiver.
- Communication is strictly allowed between CLIENT and PROPRIETAIRE.
- A user cannot message himself.
- Message content is sanitized (basic HTML tag stripping + whitespace normalization).
- Message max length is 1000 characters.
- Reservation link is optional, but when provided:
  - reservation ObjectId must be valid;
  - reservation must exist;
  - reservation must belong to the same CLIENT/PROPRIETAIRE pair in conversation.
- Only receiver can mark a message as read.
- Conversation list returns the latest message per peer + unread count.

1.2 Notification Rules
- Notifications are created for:
  - new reservation (to PROPRIETAIRE),
  - reservation cancellation (to PROPRIETAIRE),
  - payment success (to CLIENT and/or PROPRIETAIRE depending on flow),
  - new review (to PROPRIETAIRE),
  - new message (to receiver).
- Only authenticated users can access their own notifications.
- Read status is tracked per notification.

1.3 Email Verification Rules
- User account is created with isVerified=false.
- Verification token is generated, hashed, and expiration is set (1 hour).
- Login is blocked until email is verified.
- Verification endpoint validates token integrity and expiry.

1.4 User Phone Rules
- phone is required and unique.
- Accepted formats:
  - +216XXXXXXXX
  - 2XXXXXXX (7-digit local format starting from 2 to 9)
- Phone is sanitized before persistence.

1.5 Property Type Rules
- Property type is required.
- Accepted enum values:
  - MAISON
  - MAISON_DHOTES
  - VILLA
  - APPARTEMENT
  - COTTAGE
- Default value: APPARTEMENT.
- API property listing supports filtering by type.

3) SYSTEM FEATURES LIST (UPDATED)
---------------------------------

- JWT Authentication with role-based authorization (CLIENT/PROPRIETAIRE/ADMIN)
- Email verification workflow with expiring token
- Required unique phone number with Tunisian/international validation
- Property management with advanced metadata:
  - bedrooms, bathrooms, maxGuests,
  - cancellation delay (24h/48h),
  - property type classification
- Reservation workflow with overlap prevention and dynamic auto-cancellation
- Payment workflow (ONLINE + CASH confirmation) with commission tracking
- Review system bound to confirmed reservations
- Wishlist system (one per user, deduped properties)
- Notification center with read/unread management
- Secure messaging system between CLIENT and PROPRIETAIRE
- Dashboard APIs (client/proprietaire/admin)
- Image upload system (single/multiple) with type + size constraints
- Security hardening:
  - rate limiting,
  - CORS restrictions,
  - request size limit,
  - security headers,
  - core input validation
